看见 Claude Code 的运行脉搏。

Claude Runtime Monitor 不替代 Claude Code，而是在旁边建立一个本地观察面：状态、上下文、成本和 provider 余额都被整理成可读信号。敏感数据留在你的机器上。

查看源码开始接入
01

Runtime state

从 hook、statusline 与进程检测中归纳 idle、thinking、editing、waiting_permission 等状态。

02

AI core pet

AI 核心可以跟随状态变化，也支持导入 OpenPets 风格资源，让监控面板更有生命感。

03

Context pressure

读取 cc-switch 请求日志，估算上下文占用，及时暴露长会话风险。

04

Quota view

支持 DeepSeek 与通用 provider 余额检查，并保留最近快照。

清晰的数据流，少一点猜测。

Claude Codehooks 与 statusline 推送工具事件、模型、项目和上下文提示。
cc-switch本机 SQLite 日志提供请求、tokens 与成本估算。
State enginereducer 将输入归一成 MonitorSnapshot 与 RuntimeEvent。
FastifyREST、WebSocket、SQLite 持久化和 provider 余额抓取。
DashboardReact 控制台实时展示运行状态、额度、事件与成本。
接入 Claude Code
让 Claude 把运行事件推到这里

在 Claude Code 的 settings 里接入 hook 和 statusline。服务器地址保持本机 4377，本机请求会自动通过内部鉴权。

pnpm install
cp .env.example .env
pnpm dev

node scripts/claude-hooks/claude-hook.js
